## Weekly Structure Overview

### The 3-Session Model

**Total:** 6 hours/week across 3 sessions

| Session | Focus | Duration | Output |
|---------|-------|----------|--------|
| **Session A** | Learn | 2 hours | Notes + understanding |
| **Session B** | Practice | 2 hours | Labs + hands-on |
| **Session C** | Build/Write | 2 hours | Artifact (write-up/code) |

### Why This Works

```
Learn → Practice → Build

1. LEARN: Understand concepts (videos, reading)
2. PRACTICE: Apply knowledge (labs, CTFs)
3. BUILD: Create evidence (projects, write-ups)

Each week produces something tangible for your portfolio.
```

### Weekly Output Target

**Every week, produce at least ONE of:**
- Write-up (CTF solution, lab walkthrough)
- Code commit (project progress)
- Detection rule (Sigma/KQL)
- Blog draft
- Documentation update

### Scheduling Recommendations

**Option 1: Weekday Evenings**
```
Monday: Session A (Learn) - 7-9 PM
Wednesday: Session B (Practice) - 7-9 PM
Friday: Session C (Build) - 7-9 PM
```

**Option 2: Weekend Heavy**
```
Saturday: Sessions A + B - 9 AM-1 PM
Sunday: Session C - 10 AM-12 PM
```

**Option 3: Mixed**
```
Tuesday: Session A - 8-10 PM
Thursday: Session B - 8-10 PM
Saturday: Session C - 10 AM-12 PM
```

**Choose based on your energy patterns:**
- Morning person? Weekend mornings
- Night owl? Weekday evenings
- Inconsistent schedule? Block Sunday as backup

## Session Templates

### Session A: Learn (2 hours)

**Goal:** Understand new concepts deeply

```
Time Breakdown:
├── 0:00-0:10 - Review last week's notes (10 min)
├── 0:10-1:00 - Primary learning (video/reading) (50 min)
├── 1:00-1:10 - Break (10 min)
├── 1:10-1:50 - Continue learning (40 min)
└── 1:50-2:00 - Summarize key points (10 min)
```

**Session A Checklist:**
```
Before:
☐ Topic selected for this session
☐ Resources ready (videos queued, articles open)
☐ Notes app/document open

During:
☐ Take active notes (not passive watching)
☐ Pause and think about concepts
☐ Write questions that come up
☐ Note things to practice in Session B

After:
☐ 3-5 bullet summary written
☐ Practice items listed for Session B
☐ Questions noted for research
```

**Note-Taking Format:**
```markdown
# Session A Notes - [Date]
## Topic: [Topic Name]

### Key Concepts
- [Concept 1]: [Explanation]
- [Concept 2]: [Explanation]

### Questions to Research
- [ ] [Question 1]
- [ ] [Question 2]

### To Practice in Session B
- [ ] [Practice item 1]
- [ ] [Practice item 2]

### Summary (3-5 sentences)
[What I learned today...]
```

---

### Session B: Practice (2 hours)

**Goal:** Apply knowledge through hands-on work

```
Time Breakdown:
├── 0:00-0:05 - Review Session A notes (5 min)
├── 0:05-0:55 - Practice block 1 (50 min)
├── 0:55-1:05 - Break + notes (10 min)
├── 1:05-1:55 - Practice block 2 (50 min)
└── 1:55-2:00 - Document what worked/didn't (5 min)
```

**Session B Checklist:**
```
Before:
☐ Lab environment ready
☐ Practice items from Session A listed
☐ Tools open and configured

During:
☐ Focus on one topic/vulnerability
☐ Take screenshots of interesting findings
☐ Note commands that work
☐ Document errors and solutions

After:
☐ Labs completed count noted
☐ Key learnings documented
☐ Write-up topics identified for Session C
```

**Practice Log Format:**
```markdown
# Session B Practice Log - [Date]
## Topic: [Topic Name]

### Labs Completed
1. [Lab name] - [Difficulty] - [Time taken]
   - Key technique: [What worked]
   - Interesting finding: [Notable observation]

2. [Lab name] - [Difficulty] - [Time taken]
   - Key technique: [What worked]
   - Challenge faced: [Problem and solution]

### Commands/Payloads Used
```
[Useful commands with notes]
```

### For Write-up (Session C)
- Best candidate lab: [Lab name]
- Why: [Interesting technique or finding]

### Questions for Further Research
- [ ] [Question]
```

---

### Session C: Build/Write (2 hours)

**Goal:** Create portfolio artifacts

```
Time Breakdown:
├── 0:00-0:10 - Choose artifact to create (10 min)
├── 0:10-0:55 - Writing/coding block 1 (45 min)
├── 0:55-1:05 - Break (10 min)
├── 1:05-1:50 - Writing/coding block 2 (45 min)
└── 1:50-2:00 - Polish and publish (10 min)
```

**Session C Checklist:**
```
Before:
☐ Decide: write-up, code, or documentation?
☐ Template ready
☐ Session A+B notes open for reference

During:
☐ Follow template structure
☐ Include evidence (screenshots, code)
☐ Explain WHY, not just WHAT
☐ Consider reader perspective

After:
☐ Artifact published/committed
☐ Shared on LinkedIn or community (optional)
☐ Portfolio updated
```

**Artifact Decision Tree:**
```
Did I learn something new in Sessions A+B?
├── Yes → Write a concept explanation post
│
Did I solve an interesting lab?
├── Yes → Write a lab walkthrough
│
Did I build/code something?
├── Yes → Commit code + update README
│
Did I find a gap in my knowledge?
├── Yes → Document the gap + learning plan
│
None of the above?
└── Improve existing documentation or
    create a checklist/methodology doc
```

## Daily Micro-Habits

### The 10-Minute Daily Practice

Even on days without full sessions:

```
5 Minutes: Review
├── Read 1 security news item
├── OR review 5 flashcards
└── OR read 1 page of documentation

5 Minutes: Think
├── Write 1 observation
├── OR note 1 question to research
└── OR plan tomorrow's learning
```

### Flashcard System

**Tool:** Anki (free) or Notion

**Categories:**
```
1. Concepts (What is SSRF?)
2. Commands (Nmap full port scan?)
3. Payloads (Basic SQLi payload?)
4. Tools (Burp Suite shortcuts?)
5. Frameworks (MITRE ATT&CK tactics?)
```

**Sample Cards:**
```
Front: What HTTP method is used in CSRF attacks?
Back: Usually POST (form submission)

Front: Command to enumerate subdomains with subfinder?
Back: subfinder -d target.com -o subs.txt

Front: What is T1059 in MITRE ATT&CK?
Back: Command and Scripting Interpreter
```

**Review Schedule:**
- New cards: Daily
- Review cards: Spaced repetition (automatic in Anki)

### Security News Habit

**Sources (pick 1-2):**
- tl;dr sec newsletter (weekly digest)
- The Hacker News (daily headlines)
- Reddit r/netsec (community discussion)

**Process:**
```
1. Skim headlines (2 min)
2. Read 1 interesting article (3 min)
3. Note if relevant to your focus
```

## Recovery Strategies

### When You Miss a Session

**Don't:** Try to make up all missed hours

**Do:** Follow the recovery protocol

```
Recovery Protocol:

Missed 1 session:
└── Add 30 min to remaining sessions this week

Missed 2 sessions:
└── Do a condensed 3-hour session on weekend

Missed entire week:
└── Don't try to catch up
└── Do "restart week" next week
```

### Restart Week Protocol

When you've been off track for a week or more:

```
Restart Week Plan:
├── Day 1: 1 hour - Review notes, remember where you were
├── Day 3: 2 hours - Easy lab session (rebuild confidence)
├── Day 5: 2 hours - Create one small artifact
└── Day 7: 1 hour - Plan next full week
```

### Preventing Burnout

**Warning Signs:**
- Dreading study sessions
- Not retaining information
- Feeling overwhelmed by material
- Skipping sessions regularly

**Recovery Actions:**
```
1. Take complete break (3-7 days)
2. Reduce to 4 hours/week temporarily
3. Switch to easier/more fun content
4. Join a study group for accountability
5. Revisit your "why" (career goals)
```

### Maintaining Momentum

**Streaks System:**
```
Track: Days with any learning activity (even 10 min)
Goal: Build streaks of 5+ days
Reward: Small treat for 7-day streak
Reset: Don't beat yourself up, start new streak
```

**Accountability Options:**
- Study buddy (weekly check-ins)
- Public commitment (social media updates)
- Community group (null Bangalore)
- Paid accountability (coaching)

## Optimization Tips

### Energy Management

**High Energy Activities:**
- Learning new concepts
- Solving hard challenges
- Writing from scratch

**Low Energy Activities:**
- Reviewing notes
- Formatting/editing
- Organizing files

**Match to Your Schedule:**
```
If morning person:
├── Session A (Learn): Morning
├── Session B (Practice): Afternoon
└── Session C (Build): Morning (high creativity needed)

If night owl:
├── Session A (Learn): Evening (focused time)
├── Session B (Practice): Night
└── Session C (Build): Evening
```

### Environment Optimization

**Physical Setup:**
- Dedicated study space if possible
- Noise-canceling headphones
- Good lighting
- Water bottle nearby
- Phone in another room/off

**Digital Setup:**
```
Before each session:
├── Close all non-essential apps
├── Browser: only relevant tabs
├── Notifications: off
├── Music: instrumental only (if any)
└── Tools: pre-loaded and ready
```

### Pomodoro Adaptation

For 2-hour sessions:

```
Standard Pomodoro (25+5):
├── 0:00-0:25 - Focus
├── 0:25-0:30 - Break
├── 0:30-0:55 - Focus
├── 0:55-1:00 - Break
├── 1:00-1:25 - Focus
├── 1:25-1:30 - Break
├── 1:30-1:55 - Focus
└── 1:55-2:00 - Review

Or: 50/10 Blocks (better for deep work):
├── 0:00-0:50 - Deep focus
├── 0:50-1:00 - Break
├── 1:00-1:50 - Deep focus
└── 1:50-2:00 - Review
```

## Getting Started

### Your First Week

**Day 1:** Set up your schedule
```
1. Choose your 3 session times
2. Block them in calendar
3. Set reminders
4. Prepare your study space
```

**Day 2:** Prepare materials
```
1. Create notes document
2. Create weekly template
3. Set up progress tracker
4. Bookmark key resources
```

**Day 3+:** Start Session A
```
1. Follow Session A template
2. Take notes
3. Identify practice items
4. Celebrate completing first session
```

### Success Criteria

**You're on track if:**
- Completing 5+ hours/week consistently
- Producing 1+ artifact per week
- Making measurable progress on goals
- Feeling engaged (not burnt out)
- Building portfolio over time

**Adjust if:**
- Consistently missing sessions
- Not producing artifacts
- Feeling overwhelmed
- Not retaining information

---

**Remember:** Consistency beats intensity. 6 hours every week for a year beats 60 hours for one month.

Start small. Stay consistent. Build over time.
