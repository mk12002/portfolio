### Technical Interview Question Prep

#### OWASP Top 10 Questions

**Q: Explain SQL Injection and how to prevent it.**
```
Answer Framework:
1. What: User input interpreted as SQL commands
2. Impact: Data theft, auth bypass, data modification
3. Example: Login bypass with ' OR '1'='1
4. Prevention: 
   - Parameterized queries (prepared statements)
   - Input validation
   - Least privilege database accounts
   - WAF as defense-in-depth
```

**Q: What's the difference between XSS types?**
```
Answer Framework:
1. Reflected: Payload in request, reflected in response
   - Example: Search query in URL reflected on page
2. Stored: Payload saved server-side, affects all users
   - Example: Comment field storing malicious script
3. DOM-based: Client-side JavaScript manipulation
   - Example: innerHTML with user input
4. Prevention: Output encoding, CSP, input validation
```

**Q: Explain IDOR/BOLA and how you'd test for it.**
```
Answer Framework:
1. What: Access control flaw - accessing other users' objects
2. Example: Changing user ID in /api/user/123/profile
3. Testing approach:
   - Create two test accounts
   - Map all endpoints with object references
   - Swap identifiers between accounts
   - Test both read and write operations
4. Prevention: Authorization checks on every request
```

#### Cloud Security Questions

**Q: How would you secure an S3 bucket?**
```
Answer Framework:
1. Block public access (account and bucket level)
2. Bucket policy - least privilege
3. Enable encryption (SSE-S3 or SSE-KMS)
4. Enable versioning and logging
5. Use VPC endpoints for private access
6. Monitor with CloudTrail and Macie
```

**Q: Explain AWS IAM best practices.**
```
Answer Framework:
1. Least privilege - minimal required permissions
2. Use roles instead of long-term credentials
3. Enable MFA for console users
4. Regular access reviews
5. Use permission boundaries
6. No wildcards in policies when possible
7. Separate accounts for different environments
```

#### Behavioral Questions

**Q: Tell me about a security issue you found.**
```
STAR Framework:
Situation: [Context - what were you doing]
Task: [What you needed to accomplish]
Action: [Steps you took - be specific]
Result: [Outcome and impact]

Example:
"While testing an internal API during a side project, I noticed the endpoint 
returned full user objects including password hashes. I documented the issue, 
created a POC showing the risk, and proposed a fix using serialization 
controls. The fix was implemented, and I wrote a blog post about safe API 
response design."
```

**Q: How do you stay current with security?**
```
Answer:
- Subscribe to tl;dr sec and security newsletters
- Active in null Bangalore community
- Regular CTF practice on [platform]
- Follow specific researchers for my focus area
- Build projects to test new techniques
```

### Questions to Ask Interviewers

#### About the Role
```
1. What does success look like in the first 90 days?
2. What are the top 3 security challenges the team is focused on?
3. How is the security team structured?
4. What's the split between reactive and proactive work?
5. Which security tools/platforms does the team use?
```

#### About the Team
```
1. How does the security team collaborate with engineering?
2. What's the on-call rotation like?
3. How are security findings prioritized and tracked?
4. Is there budget for training and conferences?
5. What do you enjoy most about working here?
```

#### About Growth
```
1. What skills separate mid-level from senior on this team?
2. What learning opportunities are available?
3. Does the company support attending conferences like Nullcon or BSides?
4. How are projects and focus areas assigned?
```

#### Red Flags to Watch For
- No clear answer on what you'd be doing
- Security is an afterthought, not prioritized
- Very high turnover mentioned casually
- No interest in your questions

## Salary Negotiation

### Research Phase

**Sources for Bangalore Security Salaries:**
- Glassdoor (filter by Bangalore + security roles)
- LinkedIn Salary Insights
- Levels.fyi (more for MNCs)
- null community discussions (informal)
- Direct conversations with professionals

**Salary Bands (Bangalore 2024-2026):**
| Experience | Service Companies | Product Companies | MNCs |
|------------|-------------------|-------------------|------|
| 0-2 years | ₹6-12 LPA | ₹10-18 LPA | ₹15-25 LPA |
| 2-5 years | ₹12-22 LPA | ₹18-35 LPA | ₹25-50 LPA |
| 5-8 years | ₹22-35 LPA | ₹35-55 LPA | ₹50-80 LPA |
| 8+ years | ₹35-50 LPA | ₹55-80 LPA | ₹80-1.2Cr |

### Negotiation Scripts

#### When Asked Expected Salary First

```
"I'd like to understand more about the role's scope and responsibilities 
before discussing numbers. Could you share the budgeted range for this 
position?"
```

Or:

```
"Based on my research and the skills I bring, I'm looking at the range 
of [X to Y LPA]. But I'm flexible depending on the complete compensation 
picture including benefits, learning opportunities, and growth potential."
```

#### When Given an Offer

**If it meets expectations:**
```
"Thank you for the offer. I'm excited about the opportunity. Could you 
walk me through the complete compensation package including benefits, 
RSUs, and bonus structure?"
```

**If it's below expectations:**
```
"Thank you for the offer. I'm very interested in this role. Based on 
my research and the value I can bring through [specific skills/experience], 
I was expecting something in the range of [X to Y LPA]. Is there 
flexibility to move closer to that range?"
```

#### Justifying Your Ask

```
"I'm asking for [amount] because:
1. My experience with [specific skill] directly addresses [company need]
2. My project on [project] demonstrates [relevant capability]
3. My [certification] indicates verified knowledge in [area]
4. Market research shows this range for similar roles in Bangalore"
```

### Benefits to Negotiate Beyond Salary

If salary is fixed, negotiate:
- Joining bonus
- Sign-on bonus
- Higher designation/title
- More RSUs/ESOPs
- Faster first review cycle
- Conference/training budget
- Flexible work arrangement
- Certification reimbursement

### After Negotiation

**If successful:**
```
"Thank you for working with me on this. I'm excited to join and 
contribute to [Company]. Please send over the revised offer letter 
and I'll confirm my acceptance."
```

**If unsuccessful but accepting:**
```
"I appreciate you looking into this. While I was hoping for more, 
I'm still excited about the opportunity and the team. I'd like to 
accept the offer. Could we discuss a clear path to revisit 
compensation at my 6-month review based on performance?"
```
